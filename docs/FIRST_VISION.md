# 🚀 Pitch Vision

> Приложение для ведения личных заметок, таск-менеджмента и командной работы.

---

## 💡 Мотивация

Есть желание создать заметочник мечты, заточенный под собственные потребности. При этом, понимая, что это объёмная работа, — сделать его open-source.

**Побочные выгоды:**
- Проект в резюме (никогда не лишнее)
- Заработок на донатах и SaaS-решении для тех, кому такой заметочник придётся по душе

---

## ✨ Особенности и функциональность

### 📁 File-based хранилище

Все заметки, задачи, календари и т.д. хранятся в читаемом виде в `.md` и подобных файлах — наподобие Obsidian. Но в отличие от него, с более дружелюбным интерфейсом, который скрывает маркеры MD-форматирования и другие технические символы. По ощущениям — как Notion.

### 🏠 Local-first

Бесшовная офлайн-работа — в отличие от Notion, приложение полностью функционально без подключения к сети.

### 🔄 Гибкие решения синхронизации

Поддержка нескольких вариантов синхронизации, дающих гибкость в стратегии хранения и распределения заметок.

#### 📂 File-based sync

Синхронизация файлов внешними средствами. Поддерживает ограниченный набор функциональности приложения (например, нет распределения ролей и уровней доступа к заметкам). При этом максимально прост и дёшев — пользователь может взять любой инструмент синхронизации файлов: Git, Dropbox и т.д. Это просто файлы.

#### ☁️ Server-based sync

На сервере хранится мастер-копия файлов. Синхронизация происходит через него. Поддерживает полный набор функциональности.

**Варианты распространения:**
- Docker-образ для self-hosted решения
- Сервис под ключ (SaaS) платно

#### 🔗 P2P sync

Распределённая база заметок, синхронизация происходит напрямую между устройствами. Сервер выступает только оркестратором и не хранит данных.

**Варианты распространения:**
- Docker-образ для self-hosted решения
- Сервис под ключ (SaaS) платно

### 📜 Change history + Smart conflict resolving

Все файлы имеют историю изменений с diff и автором — наподобие SVN. Как и в Git, есть «закоммиченное» состояние файлов и «не закоммиченное». Синхронизация происходит только закоммиченного состояния.

В отличие от Git, история «коммитов» идёт не по веткам, а по файлам независимо — понятия веток не существует. Для пользователя это выглядит привычно: сохранённый или несохранённый файл, а также история сохранений.

При синхронизации, наподобие Git, идёт попытка слить два состояния файла. В случае конфликта пользователь должен разрешить его (на месте, в визуальном редакторе) прежде чем сохранить очередное состояние конфликтного файла.

### 🧱 Блочная структура заметок

Каждая заметка — это дерево блоков различного типа, наподобие Notion.

- **Блочные ссылки** — ссылки в заметках могут указывать не только на другую заметку (как в Obsidian), но и на конкретный блок в ней.
- **Уникальность блоков** — каждый блок уникален в рамках всех файлов. Например, при переносе блока из одной заметки в другую ссылки на него соответствующим образом обновятся.
- **Мета-свойства блоков** — каждый блок имеет технические (ID, modification date и т.д.) и опциональные семантические свойства: статус (to do, doing, done и т.д.), даты начала, завершения, дедлайна и т.д. А также кастомные свойства, настроенные пользователем. Похоже реализовано в Affinity.
- **Шаблоны** — возможность создания шаблонов заметок и шаблонов блоков.
- **Блок задачи** — встроенный блок для таск-менеджмента.
- **Блок query** — позволяет настроить фильтр блоков и вид отображения. Например, фильтр по статусу — все блоки со статусом `doing` в виде списка.

**Поддерживаемые виды отображения:**
- Список (outliner)
- Таблица
- Канбан-доска
- Водопад (timeline)
- Календарь

**Поддерживаемые свойства для фильтрации:**
- По мета-свойствам
- По тегам в блоке
- По regexp в тексте блока

### 🔙 Backlinks

Возможность отображать внизу заметки блоки, которые ссылаются на неё.

### 🔍 Smart search

Быстрый и гибкий поиск по заметкам.

### 📄 Режим документа и доски

Заметки могут иметь режим документа или доски. Каждый блок имеет координаты XYZ:
- **XY** — используется для позиционирования на доске
- **Z** — для сортировки в режиме документа

Таким образом, можно работать как с досками, так и с классическими заметками. *(Открытый вопрос: как работать с древовидностью блоков в режиме доски?)* — референс: приложение Affinity.

### 📅 Встроенный календарь

Нативно встроенный календарь, подобный Google Calendar, с возможностью создания заметок прямо из него. Отображает блоки по мета-свойствам начала и окончания.

### 👥 Роли, команды и доступы

Возможность распределения ролей, команд и настройки доступов к заметкам для авторов.

### 📱 Мобильные устройства

Поддержка мобильных и планшетных устройств с адаптацией интерфейса.

### ✏️ Совместное редактирование

Специальный режим поверх системы Change history (коммитов-сохранений), в рамках которого можно создать сессию редактирования файла онлайн в реальном времени и подключиться к ней.

В этом режиме, как например в Miro, видны другие участники (их курсоры), а их действия на лету автоматически мерджатся. Всё сохраняется на сервере в виде «не закоммиченного» изменения сессии. Синхронизация происходит с помощью CRDT-алгоритма.
